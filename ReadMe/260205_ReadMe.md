# 기본 개요
- Chord: 분산 해시 테이블(DHT)을 구현하는 대표적인 프로토콜
- Successor: 현재 노드가 담당하지 않는 키 중 자신보다 가장 가까운 다음 노드 -> ID 상으로 시계방향으로 가장 가까운 노드
- Predecessor: 현재 노드가 담당하는 키의 시작점이 되는, 자신보다 바로 앞선 노드 -> ID 상으로 반시계 방향으로 가장 가까운 노드
- Finger Table: 노드 n이 키를 빠르게 찾을 수 있도록 도와주는 라우팅 테이블
    finger[i] = successor(n + 2^(i-1)) mod 2^m,  for i = 1 to m
    노드 n, 길이는 m, ID 공간이 2^m
finger[i]는 n으로부터 2^(i-1) 떨어진 위치의 successor 노드

ex) ID 공간: 0~15, m=4 
{0, 3, 7, 10, 14}
노드 7의 successor는 10 (7 다음 노드)
노드 7의 predecessor는 3 (7 이전 노드)

노드 7의 finger[1] = successor(7 + 1) = successor(8) = 10
노드 7의 finger[2] = successor(7 + 2) = successor(9) = 10
노드 7의 finger[3] = successor(7 + 4) = successor(11) = 14
노드 7의 finger[4] = successor(7 + 8) = successor(15) = 0

> 예시(ID 공간 = 8, m = 3)
	- ID 공간(FingerTable) : 0 =< ID < 2^3, 0 ~ 7
			 0	 1	 2	 3	 4	 5	 6	 7
	- 노드 : N0, N1, N2, N3, N4, N5, N6, N7
	- 현재 노드 : N1, ID = 0
		i	Start							Interval	Successor
		1	0 + 2^(1-1) % 2^3 = 1 % 8  = 1 	[1, 2)		1 =< ID 
		2	0 + 2^(2-1) % 2^3 = 2 % 8  = 2	[2, 4)		2 =< ID 
		3	0 + 2^(3-1) % 2^3 = 4 % 8  = 4	[4, 0)		4 =< ID 
		4	0 + 2^(4-1) % 2^3 = 8 % 8  = 0		 		
		5	0 + 2^(5-1) % 2^3 = 16 % 8 = 0				
		6	0 + 2^(6-1) % 2^3 = 32 % 8 = 0	
		7	0 + 2^(7-1) % 2^3 = 64 % 8 = 0	
		
	- 현재 노드 : N1, ID = 1
		i	Start							Interval	Successor
		1	1 + 2^(1-1) % 2^3 = 2 % 8  = 2 	[2, 3)		2 =< ID 
		2	1 + 2^(2-1) % 2^3 = 3 % 8  = 3	[3, 5)		3 =< ID 
		3	1 + 2^(3-1) % 2^3 = 5 % 8  = 5	[5, 1)		5 =< ID 
		4	1 + 2^(4-1) % 2^3 = 9 % 8  = 1	[1, 2)		1 =< ID 
		5	1 + 2^(5-1) % 2^3 = 17 % 8 = 1	
		6	1 + 2^(6-1) % 2^3 = 33 % 8 = 1	
		7	1 + 2^(7-1) % 2^3 = 65 % 8 = 1
